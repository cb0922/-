========================================
    进度卡0%问题修复说明
========================================

【问题现象】
- 点击"开始爬取"后
- 进度一直显示 0%
- 状态显示"初始化中..."
- 长时间没有变化

【问题原因】
1. GUI没有解析爬虫的 [PROGRESS] 输出格式
2. Python子进程输出有缓冲，不实时
3. 缺少对爬虫启动阶段的进度更新

【修复内容】

1. 修复 app_gui.py - _parse_and_update_progress()
   - 添加对 [PROGRESS] 消息的解析
   - 识别 "Starting to crawl X URLs"
   - 识别 "Completed: X/Y (Z%)"
   - 实时更新进度条

2. 修复 app_gui.py - CrawlerWorker.run()
   - 添加 -u 参数禁用Python缓冲
   - 改进实时输出读取逻辑
   - 添加短暂休眠避免CPU过高

3. 修复 app_gui.py - start_crawl()
   - 添加立即刷新UI
   - 改善初始状态显示

【修复后的效果】

正常情况应该在5秒内开始更新进度：

时间轴：
0秒 - 点击开始
     显示："正在启动爬虫..."
     
2秒 - 爬虫启动
     日志：Starting crawler...
     显示："开始爬取 130 个网址..."
     
3秒 - 开始抓取第一个
     日志：[1/130] Crawling: ...
     进度：1/130 (0.8%)
     
5秒 - 第一个完成
     日志：[PROGRESS] Completed: 1/130 (0.8%)
     进度条更新
     状态："正在抓取第 1/130 个..."

【测试方法】

1. 重新打包程序
   cd web_crawler
   python create_portable.py

2. 启动程序，导入几个测试网址

3. 点击"开始爬取"

4. 观察：
   - 5秒内进度应该从0%开始增加
   - 状态应该变为"正在抓取第 X/Y 个..."
   - 日志区域应该显示爬虫输出

【如果仍然卡住】

请检查：
1. 日志区域是否有 "Starting crawler..." 输出
2. 命令是否正确执行
3. 是否有错误弹窗
4. 检查 urls.csv 是否存在且格式正确

【紧急处理】

如果进度卡住，可以：
1. 点击"停止"按钮中断
2. 检查网络连接
3. 减少网址数量再试
4. 查看详细日志找错误
