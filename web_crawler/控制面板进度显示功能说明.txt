========================================
    控制面板实时进度显示功能
========================================

【功能概述】

控制面板新增"抓取进度"区域，实时显示：
1. 总体进度百分比
2. 已完成/总数计数  
3. 当前操作状态
4. 最近20条抓取记录

├─────────────────────────────────────────┤
│ 正在抓取... 45.5%        58 / 130        │
│ 成功:教育部关于举办比赛的通知...        │
│                                         │

│                                         │

└─────────────────────────────────────────┘

【进度显示说明】

1. 百分比显示
   "正在抓取... 45.5%"
   - 实时计算：已完成数 / 总数 × 100%
   - 自动从 urls.csv 统计总数

2. 计数显示
   "58 / 130"
   - 左侧：已完成网址数（成功+失败）
   - 右侧：urls.csv中的总网址数

3. 当前状态
   - 显示正在进行的操作
   - 成功：显示页面标题
   - 失败：显示错误原因
   - 下载：显示PDF下载状态

4. 状态列表
   - [时间] ✓ 标题 = 成功
   - [时间] ✗ 错误 = 失败
   - 保留最近20条
   - 带时间戳

【状态标识】

✓ - 成功抓取
✗ - 抓取失败（超时/404/网络错误等）

【抓取阶段】

阶段1：初始化
  显示："初始化中..."
  进度：0%

阶段2：逐个抓取
  显示："正在抓取... XX%"
  进度：实时更新

阶段3：下载PDF（如适用）
  显示："发现 X 个PDF附件"
        "正在下载PDF文件..."

阶段4：生成报告
  显示："正在生成报告..."

阶段5：完成
  显示："抓取完成！"
  统计：成功数/失败数/总数

【完成提示】

抓取完成后弹出汇总：

┌─────────────┐
│ 爬取完成！  │
├─────────────┤
│ 统计信息：  │
│ • 总网址数: 130   │
│ • 成功: 128       │
│ • 失败: 2         │
│ • 完成率: 98.5%   │
└─────────────┘

【使用流程】

1. 准备阶段
   - 添加/导入网址到 urls.csv
   - 确认网址数量显示正确

2. 开始爬取
   - 点击"开始爬取"
   - 观察进度条和计数
   - 查看实时状态列表

3. 监控进度
   - 关注百分比变化
   - 注意失败记录（✗）
   - 查看详细日志（下方区域）

4. 完成处理
   - 查看完成统计
   - 点击"查看Word"查看结果
   - 或点击"查看报告"查看HTML

【故障处理】

情况1：进度卡在0%
→ 检查网络连接
→ 等待30秒（超时时间）
→ 或点击"停止"后重新开始

情况2：进度不更新
→ 检查urls.csv是否为空
→ 查看运行日志输出
→ 确认程序未卡死

情况3：大量失败（✗）
→ 检查目标网站是否可访问
→ 可能是网站反爬虫
→ 尝试开启动态渲染

【性能提示】

1. 130个网址预计时间：
   - 静态模式：20-30分钟
   - 动态模式：1-1.5小时

2. 进度条可能不均匀：
   - 响应快的网址先完成
   - 响应慢的后完成
   - 正常现象

3. 可中途停止：
   - 点击"停止"按钮
   - 已完成的会保存
   - 可重新开始

【技术说明】

- 进度基于 urls.csv 统计
- 每个网址无论成败都计数
- 实时解析enhanced_crawler.py的输出
- 状态列表内存只保留20条
