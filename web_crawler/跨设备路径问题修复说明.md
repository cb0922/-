# 跨设备路径问题修复说明

## 问题描述

在其他设备上运行爬虫时，GUI界面出现错误，主要原因是**工作目录不一致**导致的文件路径问题。

### 根本原因
- 本地开发时，工作目录就是程序所在目录
- 用户双击启动时，工作目录可能是桌面或其他位置
- 代码中使用了相对路径（如 `"urls.csv"`），导致找不到文件

---

## 修复内容

### 1. 启动脚本修复

#### 一键安装.bat
```batch
:: 切换到脚本所在目录（关键！确保工作目录正确）
cd /d "%~dp0"
```

#### 启动程序.bat
```batch
:: 切换到脚本所在目录（关键！确保工作目录正确）
cd /d "%~dp0"
```

**作用**：确保无论用户从何处双击启动，工作目录都是程序所在目录。

---

### 2. Python启动器修复

#### 启动器.py
```python
def main():
    # 关键：切换到脚本所在目录，确保工作目录正确
    script_dir = os.path.dirname(os.path.abspath(__file__))
    os.chdir(script_dir)
    print(f"工作目录已切换至: {script_dir}")
```

**作用**：在启动时强制切换工作目录到脚本所在目录。

---

### 3. GUI程序修复

#### app_gui.py

**添加程序根目录属性**：
```python
class MainWindow(QMainWindow):
    def __init__(self):
        super().__init__()
        # 关键：获取程序根目录（脚本所在目录）
        self.app_dir = os.path.dirname(os.path.abspath(__file__))
        print(f"[GUI] 程序目录: {self.app_dir}")
```

**所有文件操作使用绝对路径**：

| 方法 | 修复前 | 修复后 |
|------|--------|--------|
| `start_crawl` | `urls_file = "urls.csv"` | `urls_file = os.path.join(self.app_dir, "urls.csv")` |
| `load_urls` | `os.path.exists("urls.csv")` | `os.path.exists(urls_file)` |
| `load_urls_to_control` | `open("urls.csv", ...)` | `open(urls_file, ...)` |
| `quick_add_url` | `os.path.exists("urls.csv")` | `os.path.exists(urls_file)` |
| `import_national_urls` | `source_file = "全国电教馆..."` | `source_file = os.path.join(self.app_dir, ...)` |
| `import_csv` | `shutil.copy(..., "urls.csv")` | `shutil.copy(..., urls_file)` |
| `export_csv` | `shutil.copy("urls.csv", ...)` | `shutil.copy(urls_file, ...)` |
| `save_urls` | `open("urls.csv", ...)` | `open(urls_file, ...)` |

**作用**：确保所有文件操作都使用绝对路径，不受工作目录影响。

## 常见问题

### Q: 为什么本地运行正常，发送给别人就报错？
**A**: 因为本地开发时工作目录就是程序目录，而用户双击启动时工作目录可能不同。修复后强制切换工作目录到程序目录。

### Q: 路径问题有哪些症状？
**A**: 
- "URL file not found: urls.csv"
- "未找到网址库文件"
- 程序闪退
- 无法保存或加载网址

### Q: 如何验证修复是否成功？
**A**:
1. 将程序复制到桌面或其他非程序目录
2. 从该位置双击启动
3. 如果能正常读取 urls.csv 并爬取，说明修复成功
