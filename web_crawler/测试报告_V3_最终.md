## 四、发现的问题及修复

### 问题1: 禁用自动删除功能不生效
**状态:** 已修复

**描述:** 当`auto_remove_failed=False`时，`_record_failure`方法仍会返回`True`

**修复:** 在`_record_failure`方法开头添加检查：
```python
if not self.auto_remove_failed:
    print(f"  [FAIL] URL failed {fail_count} times (auto-remove disabled)")
    self._save_fail_records()
    return False
```

### 问题2: WordGenerator兼容性问题
**状态:** 已修复

**描述:** V3使用`competition_documents`字段，但WordGenerator期望`competition_pdfs`

**修复:** 更新WordGenerator支持两种字段名：
```python
# 统计
with_pdf = sum(1 for item in data if item.get('competition_pdfs') or item.get('competition_documents'))

# 目录标记
if item.get('competition_pdfs') or item.get('competition_documents'):
    pdf_run = p.add_run(" [有附件]")

# 详细信息
'pdfs': item.get('competition_documents') or item.get('competition_pdfs', [])
```

### 5.2 自动删除逻辑

```
URL访问失败
    ↓
失败次数+1，记录到文件
    ↓
检查auto_remove_failed
    ├─ False → 记录但不删除
    └─ True → 检查失败次数
              ├─ < max_retries → 继续保留
              └─ >= max_retries → 从CSV删除
```

| --proxy-file | 代理文件 | --proxy-file proxies.json |
| --max-retries | 最大重试 | --max-retries 5 |

## 八、建议

### 8.1 使用建议
1. **首次运行:** 建议启用自动删除，清理无效网址
2. **重要网址:** 如果网址重要但经常失败，使用`--max-retries 5`或`--no-auto-remove`
3. **代理配置:** 对于403网站，配置代理后重试
4. **定期备份:** 定期备份`urls.csv`和`urls_fail_records.json`
