## 根本原因

### 问题1: HTML报告生成错误（导致all模式失败）

**错误代码:**
```python
# enhanced_crawler_v3.py 第378-382行（修复前）
def generate_html_report(self) -> str:
    generator = ReportGenerator()
    report_path = generator.generate(self.results)  # 错误！generate()不接受参数
    return report_path
```

**错误原因:**
- `ReportGenerator.generate()` 方法**不接受参数**
- 但代码中传了 `self.results`
- 导致 `TypeError: generate() takes 1 positional argument but 2 were given`

**正确代码:**
```python
# 修复后
def generate_html_report(self) -> str:
    try:
        generator = ReportGenerator(self.results)  # 数据传给构造函数
        report_path = generator.generate()  # generate()不接受参数
        return report_path
    except Exception as e:
        print(f"  [WARN] HTML报告生成失败: {e}")
        return None
```

### 问题2: 文件保存路径不明确

**文档下载保存位置:**
```
./output/documents/          # 默认位置
├── PDF/                     # PDF文档

├── Excel/                   # Excel表格
├── PowerPoint/              # PPT演示
```

**Word报告保存位置:**
```

```

**HTML报告保存位置:**
```

```

## 已完成的修复

### 1. 修复 HTML报告生成 (enhanced_crawler_v3.py)

```python
def generate_html_report(self) -> str:
    """生成HTML报告"""
    try:
        generator = ReportGenerator(self.results)  # 修复：数据传给构造函数
        report_path = generator.generate()          # 修复：generate()无参数
        return report_path
    except Exception as e:
        print(f"  [WARN] HTML报告生成失败: {e}")
        return None
```

### 2. 添加错误处理

所有报告生成都添加了 try-except，防止一个环节失败导致整个程序崩溃。
