# GUI崩溃紧急修复指南

## 问题
运行GUI时出现 "Crawl failed with code 1"，大量网址失败（125/127）

## 根本原因
1. 自动删除功能在处理大量失败网址时崩溃
2. 某些网址（如青海考试院）会超时卡住
3. 失败记录累积导致问题

---

## 快速修复（3种方法）

### 方法1：一键修复（推荐）

```bash
python fix_gui_crash.py
```

然后重新启动GUI：
```bash
python app_gui.py
```

### 方法2：手动命令行（最安全）

完全绕过GUI，使用命令行：

```bash
# 设置UTF-8编码
chcp 65001
set PYTHONIOENCODING=utf-8

# 使用安全版本（已禁用自动删除）
python enhanced_crawler_v3_safe.py --urls urls.csv --no-auto-remove --output ./output
```

### 方法3：使用修复后的GUI

已更新 `app_gui.py`，默认启用安全模式：

```bash
# 清理失败记录
del urls_fail_records.json

# 启动GUI（安全模式默认开启）
python app_gui.py
```

在"设置"标签页中：
- ✅ 安全模式（推荐）- 默认已勾选
- ⬜ 自动删除失败网址 - 默认已取消

---

## 如果仍然崩溃

### 步骤1：测试单条网址

```bash
python quick_test_sample.py
```

如果这也会崩溃，说明是环境问题。

### 步骤2：分批处理

```bash
# 只处理前10个网址
python crawl_no_remove.py --max-urls 10
```

### 步骤3：排除问题网址

```bash
# 使用已分类的网址文件（排除慢速网站）
python enhanced_crawler_v3_safe.py --urls urls_normal.csv --no-auto-remove
```

## 常见问题

### Q: 为什么安全模式还会崩溃？
**A:** 请确保使用的是最新版本的 `app_gui.py` 和 `enhanced_crawler_v3_safe.py`

### Q: 如何确认是安全版本？
**A:** 查看设置标签页，"安全模式"应该默认勾选

### Q: 哪些网址会导致问题？
**A:** 主要是青海考试院 (qhjyks.com) 和 宁夏考试院 (nxjky.com)

### Q: 如何处理这些慢速网站？
**A:** 使用 `urls_normal.csv` 文件，已自动排除这些网站
