========================================
    抓取进度实时显示功能
========================================

【功能介绍】

控制面板新增实时进度显示区域，用户可以实时了解抓取进度：
1. 总体进度百分比
2. 已完成/总数计数
3. 当前抓取状态
4. 最近抓取记录列表

【界面布局】

控制台标签页
└── 抓取进度（新增区域）
    ├── 进度信息行
    │   ├── 正在抓取... 45.5%
    │   ├── 58 / 130
    │   └── 成功:教育部通知... (当前状态)
    │
    ├── 进度条 [████████░░░░░░░░░░░░] 45%
    │
    └── 实时状态列表
        [10:23:01] ✓ 教育部通知公告
        [10:23:04] ✓ 中央电化教育馆
        [10:23:07] ✓ 河北省教育厅
        [10:23:10] ✗ HTTP 404 (失败)
        [10:23:13] ✓ 山西省电化教育馆
        ...

【显示信息说明】

1. 进度百分比
   - 实时计算：已完成数 / 总数 × 100%
   - 精确到小数点后1位

2. 计数显示 (如 "58 / 130")
   - 左侧：已完成网址数
   - 右侧：总网址数
   - 自动从 urls.csv 统计

3. 当前状态
   - 显示正在进行的操作
   - 例如："正在下载PDF文件..."

4. 状态列表
   - ✓ 表示成功抓取
   - ✗ 表示抓取失败
   - 带时间戳
   - 保留最近20条记录

【状态类型】

抓取过程中可能显示的状态：

初始化阶段：
  "初始化中..."
  
抓取阶段：
  "成功: xxx通知..." - 成功抓取页面
  "失败: xxx" - 抓取失败（超时/404等）
  
下载阶段：
  "发现 X 个PDF附件"
  "正在下载PDF文件..."
  
报告阶段：
  "正在生成报告..."
  
完成阶段：
  "抓取完成！"

【完成统计】

抓取完成后显示汇总信息：

统计信息：
• 总网址数: 130
• 成功: 128
• 失败: 2
• 完成率: 98.5%

【注意事项】

1. 进度计算基于 urls.csv 中的网址总数
2. 每个网址无论成功或失败都计入进度
3. 状态列表只保留最近20条，防止过长
4. 详细日志可在下方的"运行日志"区域查看
5. 如需停止，点击"停止"按钮即可中断

【故障排查】

问题1：进度不更新
- 检查 urls.csv 是否存在且不为空
- 查看运行日志是否有输出

问题2：进度卡在0%
- 可能是第一个网址响应慢
- 等待30秒（默认超时时间）
- 或点击停止后重新开始

问题3：进度跳跃
- 正常现象，网络延迟不同
- 响应快的网址先完成
