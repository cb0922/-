## 功能介绍

### 什么是比赛通知爬虫？

这是一个专门用于自动抓取教育类网站比赛通知、公告信息的工具。它可以：

- 🌐 **批量抓取**：一次处理多个网站的通知公告
- 📄 **多格式下载**：自动下载 PDF、Word、Excel 等附件
- 📝 **智能汇总**：生成 Word 文档汇总报告
- 🎯 **比赛识别**：自动识别与比赛相关的通知
- ⏰ **定时任务**：支持定时自动爬取

### 适用场景

- 教师收集比赛信息
- 学校教务处整理通知
- 教育局信息采集
- 培训机构资料整理

## 基本操作

### 首次使用

#### 步骤1：导入网址

**方法A：单个添加**
1. 在控制台找到"网址信息源管理"
2. 输入网址（如：`https://www.moe.gov.cn/`）
3. 输入名称（如：`教育部官网`）
4. 点击"添加"

**方法B：批量导入（推荐）**
1. 准备 CSV 文件，格式如下：
   ```csv
   url,name,category
   https://www.moe.gov.cn/,教育部,国家级
   https://jyt.shanxi.gov.cn/,山西省教育厅,省级
   ```
2. 点击"导入CSV"按钮
3. 选择文件导入

**方法C：导入全国网址库**
1. 点击"导入全国网址库"按钮
2. 程序会自动导入预设的全国电教馆和教科委网址

#### 步骤2：开始爬取

1. 在控制台标签页
2. 选择爬取模式（通常选择"全部"）
3. 点击"开始爬取"按钮
4. 等待进度条完成

#### 步骤3：查看结果

- **Word报告**：点击工具栏"查看Word"或"查看最新Word文档"
- **下载的文档**：点击工具栏"查看文档"
- **HTML报告**：点击工具栏"查看报告"

### 日常使用

#### 添加新网址
1. 在控制台输入框输入网址
2. 输入名称（可选）
3. 点击"添加"按钮

#### 删除网址
1. 切换到"网址管理"标签页
2. 选中要删除的网址
3. 点击"删除选中"

#### 导出数据
1. 爬取完成后
2. 打开 `output/word_reports/` 查看Word报告
3. 打开 `output/documents/` 查看下载的文档

## 高级功能

### 1. 定时自动爬取

**设置步骤**：
1. 切换到"定时任务"标签页
2. 勾选"启用定时自动爬取"
3. 选择运行周期（每天/每周/每月）
4. 设置时间/间隔
5. 点击"保存定时设置"

### 2. 使用代理

**适用场景**：
- 爬取频繁被限制
- 需要访问特定地区网站

**设置步骤**：
1. 切换到"设置"标签页
2. 勾选"使用代理IP"
3. 选择代理类型
4. 配置代理文件或手动添加代理

### 3. 反反爬设置

程序已内置反检测功能：
- ✅ User-Agent 轮换
- ✅ 请求间隔控制
- ✅ 完整请求头模拟
- ✅ 自动重试机制

如需更隐蔽的爬取：
1. 在设置中增加请求间隔
2. 降低并发数
3. 启用代理

## 常见问题

### Q1: 为什么有些网站抓取失败？

**可能原因**：
1. 网站需要登录
2. 网站有严格的反爬机制
3. 网站使用大量 JavaScript 动态加载
4. 网络连接超时

**解决方法**：
- 勾选"启用动态渲染"（针对 JavaScript 网站）
- 增加超时时间
- 使用代理

### Q2: 如何只下载文档不生成报告？

**方法**：
1. 爬取模式选择"仅下载文档"
2. 或手动运行命令：`python enhanced_crawler_v3.py --urls urls.csv --mode doc`

### Q3: 如何只生成报告不下载文档？

**方法**：
1. 爬取模式选择"仅生成报告"
2. 或手动运行命令：`python enhanced_crawler_v3.py --urls urls.csv --mode html`

### Q4: 下载的文档在哪里？

**路径**：`output/documents/`

子目录：
- `PDF/` - PDF 文件
- `Word/` - Word 文档
- `Excel/` - Excel 表格
- `Archive/` - 压缩包

### Q5: 如何备份我的网址列表？

**方法**：
1. 在"网址管理"标签页点击"导出CSV"
2. 或直接复制 `urls.csv` 文件

### Q6: 程序突然关闭/闪退怎么办？

**排查步骤**：
1. 检查是否有杀毒软件拦截
2. 检查磁盘空间是否充足
3. 查看日志文件（如有生成）
4. 重新运行安装脚本

## 故障排除

### 问题：启动时报错 "No module named 'xxx'"

**原因**：依赖未安装完整

**解决**：
```cmd
# 重新运行安装
一键安装.bat

# 或手动安装缺失的包
pip install xxx -i https://pypi.tuna.tsinghua.edu.cn/simple
```

### 问题：爬取时全部失败

**原因排查**：
1. 检查网络连接
2. 检查 urls.csv 格式是否正确
3. 检查是否被防火墙拦截

**解决**：
1. 测试单个网址：`python test_crawl.py`
2. 检查日志输出

### 问题：下载的文档损坏/打不开

**可能原因**：
1. 下载过程中网络中断
2. 原文件本身损坏
3. 格式不支持

**解决**：
1. 删除损坏的文件，重新爬取
2. 手动访问原网址下载

### 问题：Word报告生成失败

**原因排查**：
1. 检查是否安装了 python-docx
2. 检查 output/word_reports/ 目录权限

**解决**：
```cmd
pip install python-docx -i https://pypi.tuna.tsinghua.edu.cn/simple
```

### 问题：程序占用内存过大

**解决**：
1. 减少并发数（在设置中调整）
2. 分批处理网址（每次处理20-30个）
3. 定期清理 output/ 目录
